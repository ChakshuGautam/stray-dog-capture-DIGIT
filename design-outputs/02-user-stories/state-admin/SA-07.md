# SA-07: Real-time Alerts for Unusual Activity Spikes

## User Story
**As a** State Administrator
**I want to** receive real-time alerts for unusual activity spikes and critical system events
**So that** I can respond quickly to emerging issues, potential fraud patterns, or system anomalies before they escalate

## Description
This story provides State Administrators with a comprehensive real-time alerting system that monitors all aspects of the SDCRS platform across Bihar's 38 districts. The system uses configurable thresholds, anomaly detection algorithms, and pattern recognition to identify unusual activities that require immediate attention. Alerts are prioritized by severity and can trigger automatic escalations or workflows.

## Acceptance Criteria

### Functional Requirements

#### AC1: Alert Dashboard
- **Given** I am logged in as State Admin
- **When** I access the alerts module
- **Then** I see:
  - Active alerts count by severity (Critical, High, Medium, Low)
  - Real-time alert feed with newest first
  - Alert acknowledgment and resolution status
  - Alert source distribution (system, fraud, operational, performance)
  - Time-based alert frequency chart

#### AC2: Alert Categories and Types
- **Given** the system monitors all platform activities
- **When** anomalies or threshold breaches occur
- **Then** alerts are generated for:
  - **Fraud Alerts**: Submission spikes, GPS clusters, photo similarity, coordinated behavior
  - **Operational Alerts**: MC backlog, pending verifications, resolution delays
  - **Budget Alerts**: Budget thresholds, unusual payout patterns
  - **System Alerts**: Performance degradation, integration failures, security events
  - **Quality Alerts**: Rejection rate spikes, invalid submission patterns

#### AC3: Alert Configuration
- **Given** I want to customize alert thresholds
- **When** I access alert configuration
- **Then** I can:
  - Set thresholds for each alert type
  - Configure notification channels (in-app, SMS, email)
  - Define escalation rules and timelines
  - Set quiet hours for non-critical alerts
  - Create custom alert rules with conditions

#### AC4: Alert Actions and Workflow
- **Given** I receive an alert
- **When** I view alert details
- **Then** I can:
  - Acknowledge the alert (stops escalation)
  - Assign to another administrator
  - Add investigation notes
  - Link to related alerts (cluster detection)
  - Mark as resolved with resolution category
  - Dismiss as false positive (with reason)

#### AC5: Alert Escalation
- **Given** an alert is not acknowledged within defined timeframe
- **When** escalation timer expires
- **Then**:
  - Alert escalates to next level administrator
  - Escalation notification sent via multiple channels
  - Alert priority may increase automatically
  - Escalation history is recorded

#### AC6: Alert Analytics
- **Given** I want to analyze alert patterns
- **When** I access alert analytics
- **Then** I see:
  - Alert frequency trends over time
  - Most common alert types
  - Average resolution time by category
  - False positive rate
  - Alert-to-action correlation
  - Predictive indicators

### Non-Functional Requirements
- Alert delivery within 30 seconds of trigger event
- Support for 1000+ concurrent active alerts
- Push notification delivery to mobile devices
- Alert history retention for 2 years
- 99.9% alert delivery reliability
- Support for WebSocket real-time updates

// services/AlertWebSocketManager.js

class AlertWebSocketManager {
  constructor() {
    this.ws = null;
    this.reconnectAttempts = 0;
    this.maxReconnectAttempts = 5;
    this.reconnectDelay = 1000;
    this.listeners = new Map();
    this.isConnected = false;
  }

  connect(authToken) {
    const wsUrl = `${process.env.REACT_APP_WS_URL}/alerts?token=${authToken}`;

    this.ws = new WebSocket(wsUrl);

    this.ws.onopen = () => {
      console.log('Alert WebSocket connected');
      this.isConnected = true;
      this.reconnectAttempts = 0;
      this.notifyListeners('connection', { status: 'connected' });
    };

    this.ws.onmessage = (event) => {
      try {
        const message = JSON.parse(event.data);
        this.handleMessage(message);
      } catch (error) {
        console.error('Error parsing WebSocket message:', error);
      }
    };

    this.ws.onclose = (event) => {
      console.log('Alert WebSocket disconnected', event.code);
      this.isConnected = false;
      this.notifyListeners('connection', { status: 'disconnected' });
      this.attemptReconnect(authToken);
    };

    this.ws.onerror = (error) => {
      console.error('Alert WebSocket error:', error);
      this.notifyListeners('error', { error });
    };
  }

  handleMessage(message) {
    switch (message.type) {
      case 'NEW_ALERT':
        this.notifyListeners('newAlert', message.data);
        this.playAlertSound(message.data.severity);
        break;

      case 'ALERT_UPDATE':
        this.notifyListeners('alertUpdate', message.data);
        break;

      case 'ALERT_ESCALATED':
        this.notifyListeners('alertEscalated', message.data);
        this.playAlertSound('CRITICAL');
        break;

      case 'ALERT_RESOLVED':
        this.notifyListeners('alertResolved', message.data);
        break;

      case 'SUMMARY_UPDATE':
        this.notifyListeners('summaryUpdate', message.data);
        break;

      default:
        console.log('Unknown message type:', message.type);
    }
  }

  playAlertSound(severity) {
    const soundMap = {
      CRITICAL: '/sounds/critical-alert.mp3',
      HIGH: '/sounds/high-alert.mp3',
      MEDIUM: '/sounds/medium-alert.mp3',
      LOW: '/sounds/low-alert.mp3'
    };

    const audio = new Audio(soundMap[severity] || soundMap.MEDIUM);
    audio.volume = 0.5;
    audio.play().catch(e => console.log('Audio play failed:', e));
  }

  attemptReconnect(authToken) {
    if (this.reconnectAttempts >= this.maxReconnectAttempts) {
      console.log('Max reconnection attempts reached');
      this.notifyListeners('reconnectFailed', {});
      return;
    }

    this.reconnectAttempts++;
    const delay = this.reconnectDelay * Math.pow(2, this.reconnectAttempts - 1);

    console.log(`Attempting reconnect in ${delay}ms (attempt ${this.reconnectAttempts})`);

    setTimeout(() => {
      this.connect(authToken);
    }, delay);
  }

  subscribe(eventType, callback) {
    if (!this.listeners.has(eventType)) {
      this.listeners.set(eventType, new Set());
    }
    this.listeners.get(eventType).add(callback);

    return () => {
      this.listeners.get(eventType).delete(callback);
    };
  }

  notifyListeners(eventType, data) {
    if (this.listeners.has(eventType)) {
      this.listeners.get(eventType).forEach(callback => {
        try {
          callback(data);
        } catch (error) {
          console.error('Error in alert listener:', error);
        }
      });
    }
  }

  disconnect() {
    if (this.ws) {
      this.ws.close();
      this.ws = null;
    }
    this.listeners.clear();
    this.isConnected = false;
  }
}

export default new AlertWebSocketManager();

// components/state-admin/alerts/AlertConfiguration.jsx
import React, { useState, useEffect } from 'react';
import AlertService from '../../../services/AlertService';

const AlertConfiguration = () => {
  const [config, setConfig] = useState(null);
  const [loading, setLoading] = useState(true);
  const [saving, setSaving] = useState(false);
  const [hasChanges, setHasChanges] = useState(false);

  useEffect(() => {
    loadConfiguration();
  }, []);

  const loadConfiguration = async () => {
    try {
      setLoading(true);
      const data = await AlertService.getAlertConfiguration();
      setConfig(data);
    } catch (error) {
      console.error('Error loading configuration:', error);
    } finally {
      setLoading(false);
    }
  };

  const handleConfigChange = (category, alertType, field, value) => {
    setConfig(prev => ({
      ...prev,
      [category]: {
        ...prev[category],
        [alertType]: {
          ...prev[category][alertType],
          [field]: value
        }
      }
    }));
    setHasChanges(true);
  };

  const handleNotificationChange = (category, alertType, channel, checked) => {
    setConfig(prev => ({
      ...prev,

        ...prev[category],
        [alertType]: {
          ...prev[category][alertType],
          notifications: {
            ...prev[category][alertType].notifications,
            [channel]: checked
          }
        }
      }
    }));
    setHasChanges(true);
  };

  const handleSave = async () => {
    try {
      setSaving(true);
      await AlertService.updateAlertConfiguration(config);
      setHasChanges(false);
      alert('Configuration saved successfully');
    } catch (error) {
      console.error('Error saving configuration:', error);
      alert('Failed to save configuration');
    } finally {
      setSaving(false);
    }
  };

  if (loading) {
    return <div className="loading">Loading configuration...</div>;
  }

  return (
    <div className="alert-configuration">
      <div className="config-header">
        <h1>Alert Configuration</h1>

        title="Fraud Detection Alerts"
